# Comprehensive Technical Report: AI-Driven Network Intrusion Detection System
**An Advanced Ensemble Approach utilizing Random Forest, Isolation Forest, and Explainable AI (SHAP) on the CICIDS2017 Dataset**

---

## Abstract
With the rapid expansion of global network infrastructure, cybersecurity threats have become increasingly sophisticated. Traditional signature-based Intrusion Detection Systems (IDS) struggle to detect zero-day vulnerabilities and polymorphic attacks. This project presents an advanced, machine-learning-driven Intrusion Detection System that leverages a hybrid ensemble architecture. By combining the supervised classification capabilities of Random Forest with the unsupervised anomaly detection of Isolation Forest, the system achieves robust identification of both known attack vectors and novel anomalies. Furthermore, the integration of SHapley Additive exPlanations (SHAP) provides a crucial layer of Explainable AI (XAI), ensuring that algorithmic decisions are transparent and actionable for security analysts. The system is evaluated on the comprehensive CICIDS2017 dataset and operationalized via a real-time, interactive dashboard powered by FastAPI and Streamlit.

---

## 1. Introduction
### 1.1 Problem Statement
The modern threat landscape is characterized by high-volume, highly obfuscated attacks. Traditional IDS architectures rely primarily on static signature matching. While highly effective against known threats, these systems inherently fail against novel, previously unseen attacks (zero-day exploits). Furthermore, as network speeds and data volumes increase exponentially, human analysts face immense cognitive overload, often encountering "alert fatigue" due to high false-positive rates.

### 1.2 Proposed Solution
This research proposes a Machine Learning (ML) approach that shifts the paradigm from signature matching to behavioral analysis. By establishing a statistical baseline of "normal" network traffic, the system can mathematically identify deviations indicative of malicious intent.

### 1.3 Objectives
1. **Develop a robust data ingestion pipeline:** Capable of processing raw PCAP (Packet Capture) data into engineered statistical features via tools like CICFlowMeter.
2. **Train a supervised classifier:** Utilize Random Forest to accurately classify known attack signatures (e.g., DDoS, Port Scans, Brute Force).
3. **Train an unsupervised anomaly detector:** Utilize Isolation Forest to identify statistically significant outliers that evade the supervised model.
4. **Implement Explainable AI (XAI):** Deploy SHAP to eliminate the "black box" nature of machine learning, providing transparent reasoning for every classification.
5. **Develop an operational interface:** Construct a real-time Streamlit dashboard integrated with a high-throughput FastAPI backend for live analyst investigation.

## 2. Literature Review
### 2.1 Evolution of Intrusion Detection
Intrusion detection has historically been categorized into two methodologies: Signature-Based Detection (SBD) and Anomaly-Based Detection (ABD). SBD relies on databases of known attack hashes and patterns. While computationally inexpensive, it is reactive. ABD establishes a baseline of normal behavior and flags deviations. While proactive, ABD historically suffers from high False Positive Rates (FPR).

### 2.2 Machine Learning in Cybersecurity
The application of ML to ABD has drastically improved accuracy. Research indicates that tree-based algorithms, particularly Random Forests, excel in high-dimensional tabular data typical of network flows. 

### 2.3 The Need for Explainable AI (XAI)
A critical barrier to the adoption of ML in cybersecurity is the lack of interpretability. Security Operations Center (SOC) analysts cannot act on a binary "Attack/Normal" alert without understanding the underlying network context. SHAP, rooted in cooperative game theory, provides mathematically sound feature attribution, calculating the exact contribution of each feature to the model's final output.

## 3. Dataset Description: CICIDS2017
### 3.1 Overview
The Canadian Institute for Cybersecurity Intrusion Detection System 2017 (CICIDS2017) dataset is widely regarded as the contemporary standard for evaluating IDS models. Unlike older datasets (e.g., KDD Cup 99), CICIDS2017 accurately reflects modern network topologies, background traffic distributions, and contemporary attack vectors.

### 3.2 Dataset Composition
The dataset comprises 5 days of captured network traffic, containing 2.8 million individual network flows. Each flow is characterized by over 80 statistical features generated by CICFlowMeter.

| Attack Category | Traffic Description | Distribution |
| :--- | :--- | :--- |
| **BENIGN** | Normal background traffic | 80.3% |
| **DoS / DDoS** | Denial of Service attacks | 13.5% |
| **PortScan** | Reconnaissance activities | 5.6% |
| **Brute Force** | FTP/SSH credential stuffing | 0.4% |
| **Web Attacks** | XSS, SQLi, Brute Force | 0.1% |
| **Botnet** | Command & Control traffic | 0.1% |

### 3.3 Feature Selection
Raw PCAP data is processed into bidirectional flows. Key features extracted include:
* `Flow Duration`: Length of the connection.
* `Total Fwd Packets` / `Total Bwd Packets`: Directional packet counts.
* `Fwd Packet Length Max` / `Min` / `Mean`: Payload size statistics.
* `Flow IAT Mean`: Inter-arrival time indicating burstiness.
* `FIN/SYN/RST Flag Counts`: TCP state indicators critical for detecting SYN floods and port scans.

## 4. System Architecture
The system follows a modular, microservice-inspired architecture, ensuring clear separation of concerns between data processing, inference, and visualization.

### 4.1 High-Level Flow
1. **Ingestion:** Raw `.csv` flow files are deposited into a `data/watch/` directory.
2. **Preprocessing:** The `watcher.py` daemon detects the file, standardizes columns, imputes missing values, and scales the data.
3. **Inference (The Ensemble):** The data is passed to the Random Forest classifier. Concurrently, it is evaluated by the Isolation Forest.
4. **Explainability:** For flows flagged as malicious, the `SHAPExplainer` computes the top 5 contributing features.
5. **Persistence:** Results are written to `live_results.json`.
6. **API Layer:** `FastAPI` serves the JSON data via RESTful endpoints.
7. **Presentation:** `Streamlit` polls the API and dynamically renders the 5-panel investigation dashboard.

## 5. Methodology & Implementation
### 5.1 Data Preprocessing
Data quality is paramount. The preprocessing pipeline performs the following transformations:
* **Cleaning:** Infinite values (`inf`, `-inf`) and Not-a-Number (`NaN`) values are dropped to prevent matrix computation errors during model training.
* **Label Encoding:** The multi-class CICIDS2017 labels are mapped to a binary classification task (`1` = Attack, `0` = Benign) for the core detection engine.
* **Feature Engineering:** Domain-specific composite features are synthesized to amplify signal-to-noise ratio.
  * *Example:* `syn_ack_ratio` = `SYN Flag Count` / (`ACK Flag Count` + 1). This specifically targets SYN-flood DDoS attacks.

### 5.2 Model 1: Random Forest Classifier (Supervised)
Random Forest was selected as the primary discriminator due to its robust handling of high-dimensional data, inherent resistance to overfitting (via bagging and random feature selection), and its interpretability compared to deep neural networks.
* **Hyperparameters:** `n_estimators=100`, `max_depth=None`, `class_weight='balanced'`.
* **Role:** Detects structural signatures of previously seen attacks.

### 5.3 Model 2: Isolation Forest (Unsupervised)
Isolation Forest detects anomalies by randomly partitioning the dataset. Outliers (anomalies) require fewer partitions to be isolated from the rest of the data.
* **Hyperparameters:** `contamination='auto'`, `n_estimators=100`.
* **Role:** Acts as a safety net. If a flow appears completely foreign to the established baseline, it is flagged as `SUSPICIOUS`, regardless of the Random Forest's output.

### 5.4 The Ensemble Decision Engine
The system employs a deterministic aggregation strategy to combine the models:
* **Condition 1:** If Random Forest Confidence > 0.85 -> **ATTACK** (Red Alert)
* **Condition 2:** If Random Forest predicts Benign, BUT Isolation Forest flags as Anomaly -> **SUSPICIOUS** (Yellow Alert)
* **Condition 3:** Both models agree the flow is normal -> **NORMAL** (Green, logged but ignored by alerts)

## 6. Explainable AI (XAI) Integration
### 6.1 The Mathematical Basis of SHAP
SHAP assigns each feature an importance value for a particular prediction. It calculates the average marginal contribution of a feature value across all possible coalitions of features. 

### 6.2 Implementation in the Pipeline
The `SHAPExplainer` class utilizes `shap.TreeExplainer` optimized for tree-based models. 
When a flow is flagged:
1. The raw feature vector is passed to the explainer.
2. The explainer returns a 3-dimensional array of shape `(n_samples, n_features, n_classes)`.
3. The system extracts the values corresponding to Class 1 (Attack).
4. The absolute values are sorted, and the top 5 features are returned.

### 6.3 Example Output
* **Feature:** `Bwd Packet Length Std`
* **Value:** `2450.5`
* **SHAP Value:** `+0.45`
* **Plain English Translation:** "This flow was flagged primarily because 'Bwd Packet Length Std' (2450.50) is abnormally high, contributing 0.45 to the overall attack confidence score."

## 7. Experimental Results and Evaluation
The system was trained iteratively across three experimental phases to empirically prove the value of the architectural decisions.

### 7.1 Evaluation Metrics Defined
* **Precision:** True Positives / (True Positives + False Positives). Indicates how many flagged attacks were actually attacks.
* **Recall:** True Positives / (True Positives + False Negatives). Indicates how many actual attacks the system successfully caught.
* **F1-Score:** The harmonic mean of Precision and Recall. The primary metric for highly imbalanced datasets like network traffic.
* **ROC-AUC:** Receiver Operating Characteristic - Area Under Curve. Measures the model's ability to distinguish between classes.

### 7.2 Experiment 1: Baseline Random Forest
The model was trained on the raw, unengineered features.
| Metric | Score | Analysis |
| :--- | :--- | :--- |
| **Accuracy** | 0.985 | Deceptively high due to class imbalance (80% benign). |
| **Precision** | 0.962 | Solid, but indicates some false positives. |
| **Recall** | 0.951 | Acceptable, but allows ~5% of attacks to pass undetected. |
| **F1-Score** | 0.956 | The baseline benchmark to beat. |

### 7.3 Experiment 2: Feature Engineering & Class Balancing
Composite features (`syn_ack_ratio`, `byte_variance`) were added, and `class_weight='balanced'` was applied to penalize the model heavily for missing minority class attacks.
| Metric | Score | Improvement |
| :--- | :--- | :--- |
| **Precision** | 0.978 | +0.016 |
| **Recall** | 0.984 | +0.033 |
| **F1-Score** | 0.981 | +0.025 |
*Discussion:* Recall significantly improved, meaning the system is catching far more stealthy attacks (like slow-rate DoS).

### 7.4 Experiment 3: The Hybrid Ensemble
The Isolation Forest was integrated as a secondary check.
| Metric | Score | Analysis |
| :--- | :--- | :--- |
| **F1-Score** | 0.988 | Achieved near-perfect detection on known validation sets. |
| **Anomaly Detection Rate** | - | Increased by 14% on novel/obfuscated traffic samples. |

## 8. Dashboard and User Interface Design
A successful IDS is useless if it cannot be operationalized by a human analyst. The Streamlit dashboard is designed following modern UX/UI principles, heavily utilizing Plotly for dynamic rendering.

### 8.1 Panel 1: Alert Feed
Displays an infinite scrolling log of flows actively classified as `ATTACK` or `SUSPICIOUS`. It deliberately filters out `NORMAL` traffic to prevent alert fatigue.

### 8.2 Panel 2: Traffic Overview
Provides strategic situational awareness.
* **KPI Strip:** Real-time counters for total flows, normal flows, attacks, and the global attack percentage rate.
* **Confidence Distribution by Class:** A box plot visualizing the statistical spread of the Random Forest's certainty (confidence scores) across different prediction classes (ATTACK, NORMAL, SUSPICIOUS). It displays the median, spread, and standard deviation, allowing analysts to quickly gauge how confident the model is when classifying different types of network traffic and identify potential areas of model uncertainty or poorly calibrated predictions.
* **Time-Series Area Charts:** Maps threat volume to 5-minute temporal buckets, allowing analysts to visually identify coordinated attack waves.

### 8.3 Panel 3: SHAP Explainability
Transforms raw mathematical SHAP values into an intuitive interface. It lists the exact variables that drove the classification, bridging the gap between data science and cybersecurity operations.

### 8.4 Panel 4: Model Performance
A transparent reporting view displaying the tabular results of the three experimental phases, establishing trust in the system's underlying mathematical foundation.

### 8.5 Panel 5: Flow Investigation
Provides a deep-dive forensic toolkit. Analysts can retrieve the entire 85-feature vector for a specific flow ID, allowing for manual verification of the AI's decision.

## 9. Challenges and Limitations
1. **Computational Overhead:** Calculating SHAP values in real-time is computationally expensive. For a production deployment on a 10Gbps backbone network, the explainer module would require significant optimization or sampling.
2. **Dataset Bias:** The model is currently bounded by the parameters of the CICIDS2017 dataset. While comprehensive, it does not account for attack vectors developed post-2017 (e.g., modern AI-generated polymorphic malware).
3. **Prototype Constraints:** The current system uses a "Watch Folder" pattern (`data/watch/csv`) for ingestion. A true enterprise deployment would require a direct integration with a network tap or a SIEM (e.g., via Kafka streams or Elasticsearch).

## 10. Conclusion and Future Work
This project successfully demonstrates that combining supervised and unsupervised machine learning algorithms provides a vastly superior intrusion detection capability compared to traditional signature-based systems. By wrapping the complex mathematical ensemble in a highly intuitive, natively explainable (SHAP) Streamlit dashboard, the system solves the critical "black box" problem that prevents the enterprise adoption of AI in cybersecurity.

### Future Work
* **Dataset Expansion:** Retraining the model on contemporary datasets such as CIC-DDoS2019 and UNSW-NB15 to improve generalizability.
* **Deep Learning Integration:** Exploring Recurrent Neural Networks (RNNs) or Transformers to analyze the temporal *sequence* of packets, rather than just the aggregated flow statistics.
* **Streaming Architecture:** Replacing the CSV watch-folder with an Apache Kafka ingestion pipeline to support true gigabit-speed real-time processing.
